# Lua Runtime

## Overview

The IVR system uses a **single generic Lua script** — `ivr_executor.lua` — deployed to the FreeSWITCH scripts directory. Every IVR flow, regardless of complexity, is executed by this one file. The script fetches the flow graph from the backend at call time, then iterates the graph's node list until it reaches a terminal node, the session ends, or the `MAX_STEPS` safety limit is reached.

This design means adding or changing IVR logic requires no Lua file edits — only changes to the node type registry and a redeploy.

---

## Lua Generation

`ivr_executor.lua` is generated by `backend/src/utils/luaGenerator.js`. The main export:

```js
generateIvrExecutorLua({ apiBase, apiKey, ttsEngine })
```

Parameters baked into the generated file at deploy time:

| Parameter | Source | Default |
|---|---|---|
| `apiBase` | `ENRS_API_URL` env var | `http://127.0.0.1:4100` |
| `apiKey` | `INTERNAL_API_KEY` env var | (empty — must be set in production) |
| `ttsEngine` | `FS_TTS_ENGINE` env var | `flite\|kal` |

Both `"` characters in all three values are escaped before embedding. The generation function joins an array of strings with `\n` — the resulting Lua file is a pure concatenation, not a template rendered at call time.

Node handler functions and the dispatch table are generated dynamically from `NODE_TYPE_REGISTRY` in `backend/src/nodeTypes/registry.js`. Adding a new node type to the registry and redeploying is all that is needed to add it to the Lua executor — the generation loop requires no changes.

---

## Runtime Constants

Embedded as Lua locals at the top of the generated file:

```lua
local MAX_STEPS    = 100   -- safety cap: terminates flows with accidental loops
local HTTP_TIMEOUT = 8     -- curl timeout in seconds
```

`MAX_STEPS` prevents an infinite loop in a misconfigured flow (e.g., two `goto` nodes pointing at each other). When the limit is reached, the session is hung up and a `WARN` log line is written to FreeSWITCH console.

---

## HTTP Transport

The Lua executor uses `io.popen(curl ...)` exclusively. There is no dependency on `luasocket`. This is a deliberate choice: `luasocket` is not installed by default on a fresh Debian FreeSWITCH installation, and its absence causes a fully silent call failure (`module 'socket.http' not found`) with no indication in logs.

### GET request pattern

```lua
local cmd = string.format(
  'curl -s -m %d -H "X-Internal-Key: %s" "%s" 2>/dev/null',
  HTTP_TIMEOUT, API_KEY, url)
local h    = io.popen(cmd)
local body = (h and h:read("*a")) or ""
if h then h:close() end
```

### POST request pattern

```lua
local safe_body = body_str:gsub("'", "'\\''")
local cmd = string.format(
  "curl -s -m %d -X POST -H 'Content-Type: application/json' -H 'X-Internal-Key: %s' -d '%s' '%s' 2>/dev/null",
  HTTP_TIMEOUT, API_KEY, safe_body, url)
```

The single-quote escaping in POST bodies (`gsub("'", "'\\''")`) handles shell quoting. Both GET and POST redirect stderr to `/dev/null` to prevent FreeSWITCH console noise from curl error output.

## JSON Handling

The script attempts to load `cjson` (usually bundled with `mod_lua`):

```lua
local json_ok, json = pcall(require, "cjson")
```

If `cjson` is not available, a minimal pure-Lua fallback encoder/decoder is used. The fallback handles flat string/number/boolean objects only — it does not handle nested arrays or deeply nested tables. `cjson` is always preferred and is present on all standard FreeSWITCH installations.

## Variable Interpolation

The `interp(session, str)` helper substitutes `${varname}` patterns from the FreeSWITCH channel:

```lua
local function interp(s, str)
  if not str then return "" end
  return (str:gsub("${([^}]+)}", function(k)
    return s:getVariable(k) or ""
  end))
end
```

This is used in `say` node text, `set_variable` values, `webhook` URL and body, `transfer` destination, and `condition` expected values. Any `${var}` in a node config field is resolved at call time from the current session's channel variables.

## Audio Resolution

Media library files are referenced as `/media/<filename>` URIs in node configs. The `resolve_audio(uri)` function converts these to absolute filesystem paths:

```lua
local _api       = freeswitch.API()
local SOUNDS_DIR = _api:execute("global_getvar", "sounds_dir")
                   or "/usr/share/freeswitch/sounds"

local function resolve_audio(uri)
  if not uri or uri == "" then return nil end
  if uri:sub(1,7) == "/media/" then
    return SOUNDS_DIR .. "/enrs/" .. uri:sub(8)
  end
  return uri
end
```

A `/media/welcome.wav` node config field resolves to `${SOUNDS_DIR}/enrs/welcome.wav` at call time. Direct absolute paths pass through unchanged.

## Main Execution Loop

After setup, the executor's main loop:

1. Reads the destination number: `session:getVariable("destination_number")`.
2. Calls `GET /ivr/lookup?number=<encoded_dest>` to fetch the flow graph.
3. If no flow is found, hangs up with `NO_ROUTE_DESTINATION`.
4. Iterates from `data.entry_node_id`, calling `EXEC[node.type](session, node)` at each step.
5. Each handler returns the next node ID, or `nil` to terminate.
6. Loop ends when: `cur_id` is nil, `steps >= MAX_STEPS`, or `session:ready()` returns false.
7. Final `session:hangup()` if the session is still active.

```lua
session:answer()
session:setAutoHangup(false)

local nodes  = data.nodes or {}
local cur_id = data.entry_node_id
local steps  = 0

while cur_id and steps < MAX_STEPS and session:ready() do
  steps = steps + 1
  local node = nodes[cur_id]
  local fn   = EXEC[node.type]
  local ok, next_id = pcall(fn, session, node)
  if not ok then
    freeswitch.consoleLog("ERR", "[ivr_executor] Error in node " .. cur_id .. ": " .. tostring(next_id) .. "\n")
    break
  end
  cur_id = next_id
end
```

Each handler call is wrapped in `pcall` so a runtime error in one node does not crash the entire script.

## All 17 Node Type Handlers

### `play` — Play Audio File

```lua
local f = resolve_audio(node.audio_url)
if f then s:streamFile(f) end
return node.next
```

Resolves `/media/` URIs to absolute paths. Calls `session:streamFile(path)`. Returns `node.next`.

---

### `say` — Text-to-Speech

```lua
speak(s, interp(s, node.text))
return node.next
```

The `speak(session, text)` helper calls `session:execute("speak", TTS_ENGINE .. "|" .. text)`. The `TTS_ENGINE` constant is `"engine|voice"` format (e.g., `"flite|kal"` or `"polly|Joanna"`).

---

### `gather` — Collect DTMF Digits

```lua
local digits
local pf = resolve_audio(node.prompt_audio_url)
if pf then
  digits = s:playAndGetDigits(1, max_d, 3, timeout, terms, pf, "", "[0-9#*]+", "", 0)
else
  if pt ~= "" then speak(s, pt) end
  digits = s:getDigits(max_d, terms, timeout)
end
s:setVariable(node.variable_name or "gather_result", digits or "")
return br[digits] or br["_default"] or br["invalid"]
```

`timeout` is `node.timeout_seconds * 1000` (milliseconds). `terminators` defaults to `"#"`. Branch routing: `br[digits]` → `br["_default"]` → `br["invalid"]`. The `_default` branch catches any input not matched by an explicit digit key.

---

### `condition` — Branch on Variable

Reads `session:getVariable(node.variable)` and compares against `interp(s, node.expected_value)`.

Supported operators:

| Operator | Behaviour |
|---|---|
| `==` | Exact string match |
| `!=` | Inequality |
| `contains` | Lua `string.find(exp, 1, true)` |
| `starts_with` | `val:sub(1, #exp) == exp` |
| `ens_pin_valid` | `POST /ens/verify-pin { trigger_number, pin }` → if `authorized`, also calls `GET /ens/lookup` and stores `ens_configuration_id` + `ens_blast_clid` as session variables |
| `ens_callback_valid` | `GET /ens/callbacks/authorize?reply_clid=&caller=` → if `authorized`, stores `ens_notification_uuid`, `ens_recording_file`, `ens_delivery_id` |
| `time_of_day` | `expected_value` format: `HHMM-HHMM`. Supports overnight ranges (e.g., `2200-0600`). Compared against `os.date("%H%M")`. |
| `day_of_week` | `expected_value`: comma-separated day numbers `0-6` (0=Sun). Compared against `os.date("%w")`. |

Returns `node.true_node` if condition is met, `node.false_node` otherwise.

---

### `goto` — Jump to Node

```lua
local function exec_goto(s, node) return node.target_node_id end
```

The simplest handler. The dispatch table key uses the string syntax `["goto"]` because `goto` is a reserved word in Lua 5.2+:

```lua
local EXEC = {
  ["goto"] = exec_goto,
  -- other types ...
}
```

This is the only per-type special-casing in the dispatch table generator.

---

### `ens` — Trigger ENS Blast

Resolves `configuration_id` from `node.ens_configuration_id` or falls back to `session:getVariable(node.ens_config_var)`. Calls `POST /ens/notifications { configuration_id, triggered_via: "PHONE", caller_number, recording_file }`. Stores `ens_notification_uuid` as a session variable on success. Returns `node.next` (optional — ENS can be a non-terminal node).

---

### `ers` — Trigger ERS Conference (Simple)

Generates a conference room name: `"ers_" .. tostring(cfg_id) .. "_" .. tostring(os.time())`. Calls `POST /ers/incidents { configuration_id, caller_number, conference_room, group_type }`. Bridges the caller with `session:execute("conference", room .. "@default")` — this call **blocks** until the caller leaves. After the caller leaves, calls `POST /ers/incidents/:uuid/complete`. Returns `nil` (terminal node).

---

### `hangup` — End the Call

Optionally plays a goodbye audio file, then calls `session:hangup()`. Returns `nil`.

---

### `record_message` — Record Caller Audio

Resolves the recording directory from `node.record_dir` or `_api:execute("global_getvar", "recordings_dir") .. "/ivr"`. Generates a file path: `rec_dir .. "/ivr_" .. session_uuid .. "_" .. os.time() .. ".wav"`. Plays a prompt, plays a beep tone (`tone_stream://%(500,0,640)`), and calls `session:recordFile(fpath, max_seconds, silence_threshold, silence_hits)`. Stores the path in `session:setVariable(node.variable_name, fpath)`.

---

### `set_variable` — Set Session Variable

```lua
s:setVariable(node.variable or "unknown_var", interp(s, node.value) or "")
return node.next
```

Useful for pre-populating session variables before downstream condition nodes or ENS triggers.

---

### `transfer` — Transfer Call

```lua
s:execute("transfer",
  dest .. " " .. (node.dialplan or "XML") .. " " .. (node.context or "default"))
return nil
```

Hands off call control to the FreeSWITCH dialplan. Terminal — no `next` node because the transferred dialplan takes over.

---

### `webhook` — Fire HTTP POST

Constructs and fires a curl POST to an arbitrary external URL (not routed through `API_BASE`/`API_KEY`):

```lua
local cmd = string.format(
  "curl -s -m %d -X POST -H 'Content-Type: application/json' -d '%s' '%s' 2>/dev/null",
  HTTP_TIMEOUT, safe_body, url)
local h = io.popen(cmd)
if h then h:close() end
return node.next
```

Fire-and-forget — the response is not read. Returns `node.next`.

---

### `ers_ring_all` — Ring All Tier Responders

Calls `POST /ers/ring-all { configuration_id, tier, caller_number, caller_name }`. On success, receives `d.conference_room` and `d.incident_uuid`. Bridges the caller with `session:execute("conference", d.conference_room .. "@default")` — blocks until the caller leaves. Then calls `POST /ers/incidents/:uuid/complete`. On failure (no responders, backend error): speaks an audible error message and hangs up with `NORMAL_CLEARING`.

The backend ring loop runs in parallel during the entire time the caller is in the conference room, continuously re-ringing unansered responders.

---

### `ers_overflow_check` — Route by Live Tier Occupancy

Calls `GET /ers/tier-status?configuration_id=<id>`. Returns tier occupancy based on live FreeSWITCH conference member count (never the DB status column). Routes to:
- `br["primary"]` if Level 1 tier is not occupied.
- `br["secondary"]` if Level 2 tier is not occupied.
- `br["full"]` if both tiers are occupied.

On API failure, falls through to `br["full"]`.

---

### `ers_overflow_wait` — Hold in Queue

Calls `POST /ers/overflow/enqueue { configuration_id, caller_number, caller_name, destination_number }`. Receives `enq.queue_id`. Plays a hold prompt (audio file or TTS). Then polls `GET /ers/overflow/poll?queue_id=<id>` every 3 seconds (via `session:execute("playback", "silence_stream://3000")`). When `d.ready` and `d.conference_room` are set, bridges the caller into the conference. When `d.cancelled`, routes to `node.next` (fallback). When `max_wait_seconds` is reached, routes to `node.next`.

---

### `ens_blast_record` — PIN Gate + Record + Blast

Complete blast trigger in one node:

1. **PIN Gate**: loops up to 3 times. Collects up to 8 digits terminated by `#`. Calls `POST /ens/verify-pin`. On third failure, speaks "Maximum authorization attempts exceeded" and hangs up with `CALL_REJECTED`.
2. **Resolve config**: uses `node.ens_configuration_id` if set; otherwise calls `GET /ens/lookup?number=<dest>`.
3. **Record**: generates path `<recordings_dir>/ens/ens_<cfg_id>_<epoch>.wav`. Plays record prompt, beep tone, then `session:recordFile(fpath, max_record_seconds or 120, 500, 3)`.
4. **Blast**: calls `POST /ens/notifications { configuration_id, triggered_via: "PHONE", caller_number, recording_file: fpath }`.
5. Speaks confirmation or error. Returns `node.next`.

---

### `ens_playback_gate` — Authorized Caller Playback

Calls `GET /ers/playback/authorize?configuration_id=<id>&caller=<callerNum>`. If `d.authorized`:
- If `d.recording_file` is set: `session:streamFile(d.recording_file)`.
- Else: speaks `node.no_message_text` (or default "There is no active emergency message at this time.").
- Returns `node.true_node`.

If not authorized: logs a WARN with the rejection reason. Returns `node.false_node`.

## Session Variables Summary

Key session variables used across node types:

| Variable | Set by | Read by |
|---|---|---|
| `destination_number` | FreeSWITCH (built-in) | All nodes needing the dialed number |
| `caller_id_number` | FreeSWITCH (built-in) | `ens`, `ers`, `ers_ring_all`, `ens_blast_record` |
| `caller_id_name` | FreeSWITCH (built-in) | `ers_ring_all` |
| `ens_configuration_id` | `condition (ens_pin_valid)` | `ens` node (`ens_config_var` field) |
| `ens_blast_clid` | `condition (ens_pin_valid)` | Downstream ENS origination |
| `ens_notification_uuid` | `ens`, `ens_blast_record` | Audit / downstream reference |
| `ens_recording_file` | `condition (ens_callback_valid)` | Downstream playback |
| `ens_delivery_id` | `condition (ens_callback_valid)` | Downstream reference |
| `ers_incident_uuid` | `ers`, `ers_ring_all` | Downstream reference |
| `recorded_file_path` | `record_message` (via `variable_name`) | `ens`, `ers` nodes |
| `gather_result` | `gather` (default `variable_name`) | `condition` |
